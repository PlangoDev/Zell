# BrainFormer v15 — what's new, and how to run it

v15 turns the research into shipping code. Everything is **flag-gated and default-off**,
so the proven v3 path is unchanged. Build as before (`make`); the new modules are
header-only (`ngram.h`) or pure host code.

## The honest frame
The ppl ~2000→17 gap is **structural** (parallel-expert mixture, fixed random codes,
linear readout) and no single graft crosses it. v15's realistic target is **lower ppl +
unlocking scale**, not a coherent chatbot. The biggest wins here are (1) making *more
data help again* and (2) the infini-gram interpolation.

---

## 1. Fix "more data hurts" — LR annealing + weight decay  ★ run this first
The 500M run regressed vs 50M (static 3412→3703): constant LR over-trains and fixed
weight-decay over-regularizes at scale.

```
./brain --meta meta.json --train-tokens 500000000 \
  --n-classes 256 --n-layers 4 --multiscale 1 --neg-word 8 --adapt 1 \
  --lr-final 0.1 --wd 0
```
- `--lr-final F` cosine-anneals lr → lr·F over the run (default 1.0 = old behavior).
- `--wd`, `--decay-every` now CLI-exposed (were hardcoded 1e-5 / 200).

**Keep/kill:** static ppl < 3412 ⇒ scaling recovered ⇒ the v14 6B overnight run is greenlit.

## 2. Homeostatic load-balancing  (`--balance`)
DeepSeek's aux-loss-free controller on the per-granule selection bias (`gbias`), which
was scaffolded but **always zero** — there was no load-balancing at all. Recovers the
capacity behind the 0.348 usage-Gini and flattens the hot-granule tail.
```
... --balance 1 --balance-lr 0.1 --balance-every 200
```
Self-correcting negative feedback (over-firing granule → higher bias → suppressed).
Dual-GPU safe (controller state reset at each sync). Incompatible with `--gran-frac<1`
(auto-disabled). Tune `--balance-lr` up if the Gini barely moves on short runs.

## 3. Infini-gram interpolation  (`--ngram`)  ★ highest quality EV
A suffix array over the corpus gives the next-token distribution from the **longest
matching suffix**; we mix it into eval: `P = (1-λ)·P_brain + λ·P_ngram`. Web-verified
~42% relative ppl cut when interpolated with a neural LM. Zero training, no backprop.

**Build the index once** (internet-on builder notebook, `pip install pydivsufsort`):
```
python tools/build_ngram.py --meta meta.json      # writes tokens.bin.sa (+ .json)
```
**Then eval with it:**
```
./brain --meta meta.json ... --adapt 1 --ngram /kaggle/working/tokens.bin --ngram-lambda 0.3
```
Prints `Brain + infini-gram` and `Brain ADAPTIVE + infini-gram` lines vs the raw brain.
Sweep λ ∈ {0.1, 0.2, 0.3, 0.5}. The C query is byte-for-byte validated against a Python
reference (`build_ngram.py --selftest`).

## 4. Checkpoint / resume  (`--save` / `--load`)  — for the 12h overnight limit
Saves only the **learned** state (gwt, gbias, Wc, Ww, biases, mixture ≈ 0.4 GB); the
fixed-random front-end is regenerated from the seed, so `--load` skips competitive
feature learning and resumes instantly.
```
./brain --meta meta.json --train-tokens 3000000000 ... --save ckpt.bin     # session 1
./brain --meta meta.json --train-tokens 3000000000 ... --load ckpt.bin --save ckpt.bin  # session 2: train more
./brain --meta meta.json --train-tokens 1 ... --load ckpt.bin --adapt 1 --ngram ...      # load + eval only
```
Header-validated against config+corpus (refuses to load a mismatched checkpoint).

## Deliberately NOT built (and why — these were on the research list)
- **MTP (multi-token prediction):** doesn't fit our architecture. Our layers are
  *parallel experts* combined by a mixture, not a shared-trunk hierarchy — making layer
  *l* predict token +h(l) just makes it useless for next-token (no backprop to share the
  gain). The `--boost` flag already provides a forward cross-layer cascade.
- **DeltaNet true forgetting:** the correct full-row decay roughly **doubles** the
  word-head scatter (our bottleneck) for marginal gain over `--wd` + `--lr-final`.
- **IVF granule routing:** the "skip computing all G granules" premise fights our
  **tensor-core** dense GEMM (skipping granules loses tensor-core throughput), so the
  realistic win is ~break-even, not the >5× the research assumed for non-tensor setups.
  Needs Nsight profiling on the T4 before it's worth building.
- **DFA (fake backprop):** the cross-layer credit it adds is conditioning-only
  (web-verified DFA-on-LM is 93 vs 30 ppl) and `--boost` already covers it.

## Suggested A/B order (50M, keep/kill by val ppl)
1. `--lr-final 0.1 --wd 0` (does scaling work again?) → greenlights the big run
2. `+ --balance 1` (does load-balancing help?)
3. build the suffix array → `+ --ngram ... --ngram-lambda {0.1..0.5}` (the big quality swing)
4. `--k-active {128,256}` (research: K=64 is likely too sparse for language)
